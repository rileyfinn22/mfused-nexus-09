




# Public Vendor Shipment Update Link -- Implementation Plan

## Overview
Allow admins to generate a shareable, no-login-required link from the Production page. Vendors/forwarders open the link and see a spreadsheet-style table where they can fill in carrier, tracking/PRO numbers, ETAs, and notes for shipment legs.

## Step 1: Database Migration

Create the `shipment_share_links` table, RLS policies, and two SECURITY DEFINER RPC functions in a single migration.

**Table: `shipment_share_links`**
- id (uuid PK), token (text, unique, 64-char hex), company_id (uuid), created_by (uuid), order_ids (uuid[]), expires_at (timestamptz, default now()+30 days), is_active (boolean, default true), label (text), created_at (timestamptz)

**RLS Policies:**
- Vibe admins: full CRUD
- Anonymous SELECT where token matches (for the public page to load link metadata)

**RPC: `get_shipment_legs_by_token(p_token text)`**
- Validates token is active and not expired
- Returns shipment legs joined with order_number for matching order_ids
- SECURITY DEFINER, no auth required

**RPC: `update_shipment_leg_public(p_token text, p_leg_id uuid, p_carrier text, p_tracking_number text, p_estimated_arrival timestamptz, p_notes text)`**
- Validates token, checks leg belongs to an order in the token's order_ids
- Updates only carrier, tracking_number, tracking_url, estimated_arrival, notes
- Auto-generates tracking_url from carrier + tracking_number using SQL CASE
- Returns JSON success/error

## Step 2: New Page -- `src/pages/ShipmentUpdate.tsx`

A public page (no DashboardLayout, no auth required) at `/shipment-update?token=xxx`.

**Behavior:**
- Reads token from URL query params
- Calls `get_shipment_legs_by_token` RPC to load data
- Displays VibePKG branding header (logo + title)
- Renders a spreadsheet-style table with columns: Order #, Leg #, Type, Origin, Destination, Carrier (editable input), Tracking # (editable input), ETA (editable date input), Notes (editable textarea), Status (read-only badge)
- Tracks modified rows in local state
- "Save All Changes" button calls `update_shipment_leg_public` for each modified row
- Toast feedback on success/error
- Shows expired/invalid token message when appropriate

## Step 3: New Component -- `src/components/GenerateShipmentLinkDialog.tsx`

A dialog opened from the Production page bulk action bar.

**Features:**
- Optional label input (e.g. "Feb Shipment - Factory A")
- Expiration date picker (defaults to 30 days from now)
- On submit: inserts into `shipment_share_links` with token generated via `encode(extensions.gen_random_bytes(32), 'hex')`
- Shows the generated link with a copy-to-clipboard button
- Uses the published URL as base: `https://vibepkgportal.lovable.app/shipment-update?token=...`

## Step 4: Modify `src/App.tsx`


## Step 5: Modify `src/pages/Production.tsx`

Add a "Share Link" button to the existing bulk action bar (next to "Add Shipping Leg"):
- Opens `GenerateShipmentLinkDialog` with the selected order IDs
- Import the new dialog component and add state for its open/close

## Technical Details

### Security
- Tokens are 64-char random hex (cryptographically secure, generated by Postgres)
- All public data access goes through SECURITY DEFINER functions that validate the token
- No direct table access for anonymous users on `shipment_legs`
- Vendors can only update logistics fields -- not status, dates, or leg structure
- Links expire after 30 days by default and can be deactivated by admins
